3 — Routing & URL design

Use clean, SEO-friendly routes:

/services/{serviceSlug}/{locationSlug}   -> /services/plumbing/portsmouth
/services/{serviceSlug}                  -> service landing
/locations/{locationSlug}                -> location landing

Use attribute routing for clarity and control. (Docs: attribute routing & endpoints).  ￼

Program.cs (minimal example)

var builder = WebApplication.CreateBuilder(args);
builder.Services.AddControllersWithViews();
builder.Services.AddDbContext<AppDbContext>(options => 
    options.UseNpgsql(builder.Configuration.GetConnectionString("Default")));

var app = builder.Build();
app.UseStaticFiles();
app.UseRouting();
app.MapControllerRoute(
    name: "serviceLocation",
    pattern: "services/{serviceSlug}/{locationSlug}",
    defaults: new { controller = "Service", action = "Location" });
app.MapControllerRoute(
    name: "service",
    pattern: "services/{serviceSlug}",
    defaults: new { controller = "Service", action = "Index" });
app.MapDefaultControllerRoute();
app.Run();

4 — Controller example

public class ServiceController : Controller
{
    private readonly AppDbContext _db;
    private readonly SeoService _seo;

    public ServiceController(AppDbContext db, SeoService seo) { _db = db; _seo = seo; }

    [HttpGet]
    public async Task<IActionResult> Location(string serviceSlug, string locationSlug)
    {
        var page = await _db.ServiceLocationPages
            .Include(p => p.Service).Include(p => p.Location)
            .FirstOrDefaultAsync(p => p.Service.Slug == serviceSlug && p.Location.Slug == locationSlug);

        if (page == null) return NotFound();

        _seo.SetMeta(
            title: page.TitleTag ?? $"{page.Service.Title} in {page.Location.Name}",
            description: page.MetaDescription,
            canonical: page.CanonicalUrl ?? Url.Action("Location", "Service", new { serviceSlug, locationSlug }, Request.Scheme)
        );

        var jsonLd = _seo.BuildLocalBusinessJsonLd(page); // inject structured data
        ViewData["JsonLd"] = jsonLd;

        return View(page);
    }
}

The SeoService centralises meta/canonical/JSON-LD generation. Keep canonical logic consistent to avoid duplicate content problems (canonical best practices).  ￼

6 — JSON-LD LocalBusiness (generate per location/service)

Use Google’s recommended LocalBusiness JSON-LD schema for local SEO. Include accurate address, telephone, areaServed and url. Use AreaServed to list towns/counties you serve. (Google docs + Schema.org).  ￼

Example generator (simplified):

public string BuildLocalBusinessJsonLd(ServiceLocationPage page)
{
    var ld = new {
        @context = "https://schema.org",
        @type = "LocalBusiness",
        name = $"{page.Service.Title} — {page.Location.Name}",
        url = page.CanonicalUrl ?? UrlForPage(page),
        telephone = "+44 1234 567890",
        address = new {
            @type = "PostalAddress",
            addressLocality = page.Location.Name,
            addressRegion = page.Location.County,
            addressCountry = "UK"
        },
        areaServed = new [] { page.Location.Name, page.Location.County }
    };
    return JsonSerializer.Serialize(ld, new JsonSerializerOptions { WriteIndented = false });
}

7 — Programmatic sitemap generation

Sitemap must include every (service, location) page. Use a library (DotNetSitemapGenerator / Sidio.Sitemap) or write a simple endpoint that outputs XML from DB rows. See established samples and NuGet packages.  ￼

Simple controller action for /sitemap.xml:

[Route("sitemap.xml")]
public async Task<IActionResult> Sitemap()
{
    var pages = await _db.ServiceLocationPages.Include(p=>p.Service).Include(p=>p.Location).ToListAsync();
    var sb = new StringBuilder();
    sb.AppendLine(@"<?xml version=""1.0"" encoding=""utf-8""?>");
    sb.AppendLine(@"<urlset xmlns=""http://www.sitemaps.org/schemas/sitemap/0.9"">");
    foreach(var p in pages) {
        var url = Url.Action("Location","Service", new { serviceSlug = p.Service.Slug, locationSlug = p.Location.Slug }, Request.Scheme);
        sb.AppendLine("<url>");
        sb.AppendLine($"  <loc>{System.Net.WebUtility.HtmlEncode(url)}</loc>");
        sb.AppendLine($"  <lastmod>{p.LastUpdated:yyyy-MM-dd}</lastmod>");
        sb.AppendLine("</url>");
    }
    sb.AppendLine("</urlset>");
    return Content(sb.ToString(), "application/xml", Encoding.UTF8);
}

8 — Making many unique pages at scale (content strategy in code)

Don’t just duplicate a template replacing the town name. Instead:
	•	Store reusable snippets (intro, features, testimonials) and pick 2–3 per page programmatically.
	•	Allow manual overrides per ServiceLocationPage for H1, meta, and body.
	•	Use a ContentRenderer to stitch snippets into the final HTML and inject local facts (e.g., nearest train station, case study lines).
	•	Track LastUpdated to show freshness.

This reduces thin/duplicate content while scaling. Tools and plugins exist to help with snippet management; implement a lightweight WYSIWYG in the admin area for exceptions.
